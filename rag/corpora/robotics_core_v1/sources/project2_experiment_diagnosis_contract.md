# 项目二实验诊断 Agent 设计契约

## 工具链顺序

诊断流程先查询实验日志和运行元数据，再用 Python 计算距离矩阵与派生指标，检测满足公开阈值的异常
时间窗，最后关联同一时间轴上的机器人控制指令、状态变化与恢复行为。RAG 提供版本匹配的 ROS 文档、
项目接口和历史失败，但不直接制造实验事实。

## 事实、假设与未知

报告必须区分 observation、derived metric、hypothesis 和 unknown。抖动原因只能表述为由时间窗口、
日志字段、控制指令和文档共同支持的可能原因；没有证据时不得把相关性写成确定因果。

## 报告证据

最终图表和报告记录输入 artifact hash、分析参数、异常阈值、Trace id、工具输出与 RAG source id。
发布报告属于外部副作用，需要人工审批；读取日志和运行 Python 分析保持只读。
